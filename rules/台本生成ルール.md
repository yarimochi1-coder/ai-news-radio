# AIニュースラジオ 台本生成ルール

## 番組概要
- **番組名**: 「AIニュースラジオ」
- **尺**: 約15分（日本語で約4,500〜5,000文字）
- **形式**: 一人語りポッドキャスト風
- **リスナー**: 有持雄陽さん（非エンジニア、AIを業務に高度活用中）
- **配信頻度**: 毎朝（通勤時間に聴く想定）

## リスナープロフィール
- 外壁塗装会社のマーケ・開発担当としてAIを活用
- Claude API / Claude Code / LINE Bot / GitHub Actions / WordPress APIなどを実務で使用
- プログラミングは素人だが、AIツールの活用には非常に詳しい
- 関心: AIの最新動向、ビジネス活用事例、新サービス・新モデルのリリース

## トーン・スタイル
- カジュアルで親しみやすいポッドキャスト風
- 「おはようございます！」で始める
- 敬語ベースだがフレンドリー
- 難しい技術用語は必ず噛み砕いて説明
- 「これ、有持さんの業務だと〜に使えそうですよね」のような実務との接点を入れる
- 語りかけ口調（「〜なんですよね」「〜じゃないですか」）

## 台本構成

### 1. オープニング（約1分）
```
おはようございます！AIニュースラジオ、{日付}の放送です。
今日は{ニュース件数}つのトピックをお届けします。
{今日のラインナップを簡単に紹介}
```

### 2. メインニュース（3〜4本、各約3分）
各ニュースの構成:
1. **見出し**: ニュースの一言要約
2. **何が起きたか**: 事実を簡潔に
3. **なぜ重要か**: 業界へのインパクト
4. **有持さんへの示唆**: 「これ、みのりの〜にも応用できそうですよね」等

### 3. エンディング（約1分）
```
以上、今日のAIニュースラジオでした。
今日のポイントをまとめると...{3行要約}
それでは、今日も良い一日を！
```

## ニュース選定基準
### 優先度高
- LLM（Claude, GPT, Gemini等）の新モデル・新機能リリース
- AI活用のビジネス事例（特に中小企業・マーケティング）
- SEO/検索エンジンとAIの関連ニュース
- API・開発ツールの新機能（非エンジニアでもわかるレベルで）
- AI規制・法律の動き

### 優先度中
- AI業界の資金調達・M&A
- 研究論文の画期的な成果（実用面を中心に）
- AI関連の統計データ・調査結果

### 除外
- 論文の詳細な技術解説（数式・アーキテクチャの詳細）
- ゲーム・エンタメ寄りのAI活用（業務と関連薄い場合）
- 海外の政治的なAI議論（日本のビジネスに影響薄い場合）

## 出力フォーマット
- マークダウン形式で出力
- ファイル名: `{YYYY-MM-DD}_台本.md`
- 読み上げ用テキスト（TTS用）は別途 `{YYYY-MM-DD}_読み上げ.txt` に出力
  - マークダウン記法を除去
  - 英語の固有名詞にはカタカナ読みを付与（例: Claude → クロード）
  - 数字は日本語読みに変換

## 検索クエリ例

- "Google Gemini update"
- "AI business automation"
- "AI SEO marketing"
